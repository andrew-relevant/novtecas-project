# ТЗ на разработку сайта Novtecas (версия для ИИ-исполнителя)

## 1. Контекст проекта

- Проект: корпоративный сайт производителя дорожных материалов Novtecas.
- Текущий стек и инфраструктура уже настроены: `Docker`, `Strapi`, `Next.js`, `PostgreSQL`.
- Цель документа: дать ИИ-разработчику однозначное, структурированное ТЗ для реализации MVP без лишних уточнений.
- Старый сайт используется как референс структуры/контента: `https://www.novtecas.ru/`.
- Принцип редизайна: сохраняем узнаваемую информационную логику старого сайта, но реализуем современный UI/UX, упрощенную админку и улучшенное SEO.

## 2. Цели и KPI

### Бизнес-цели

- Презентация компании и продукции.
- Сбор заявок на покупку и дилерство.
- Упрощение обновления контента через CMS.

### Продуктовые KPI (MVP)

- Форма "Купить" и форма "Заявка на дилерство" отправляют заявки без ошибок.
- Все ключевые страницы индексируемы (SSR/SSG, мета-теги, sitemap, robots).
- Контент управляется через админку Strapi (без правок кода для редактора).

## 3. Границы MVP

### Входит в MVP

- Многостраничный сайт с разделами: Компания, Продукция, Портфолио, Дилерам, Медиа, Контакты.
- Каталог товаров + карточка товара.
- Калькулятор расхода.
- Интеграция с картой Яндекс для дилеров.
- Формы обратной связи.
- Галереи фото/видео (Swiper).
- SEO-база (meta, OpenGraph, sitemap, robots, canonical).

### Не входит в MVP (вынести в backlog)

- Полноценная e-commerce корзина и онлайн-оплата.
- Личный кабинет пользователя.
- Мультиязычность.
- Сложные интеграции CRM/1С (только если появится отдельное ТЗ).

## 4. Технические ограничения и стандарты

### Технологии

- Frontend: `Next.js` (App Router), `TypeScript`, `shadcn/ui`, `lucide-react`.
- Backend/CMS: `Strapi`.
- DB: `PostgreSQL`.
- Контейнеризация: существующий `docker-compose`.

### Общие требования к реализации

- Чистая компонентная архитектура.
- Строгая типизация (`TypeScript` без `any` в доменной логике).
- Формы с валидацией на клиенте и сервере.
- Обработка ошибок с понятными сообщениями пользователю.
- Контент и SEO-данные управляются через CMS.

## 7.9 Калькулятор расхода (встраиваемый компонент)

Переиспользуемый компонент, встраивается на страницу "Продукция" и в карточку товара.

- **Входные поля**:
  - Длина участка (м).
  - Ширина участка (м).
  - Глубина выбоины / толщина слоя (см).
- **Результат** (расчет):
  - Объем (м³) = длина × ширина × (глубина / 100).
  - Масса (кг) = объем × плотность (по умолчанию ~2000 кг/м³, уточнить у заказчика).
  - Количество мешков = масса / вес_мешка (30 или 50 кг, выбирается пользователем).
- **UX**: результат обновляется в реальном времени при вводе; кнопка "Заказать расчетное количество" передает данные в форму "Купить".
- Референс: калькулятор со старого сайта https://www.novtecas.ru/produkty/holodnyj-asfalt/

## 8. Формы и лидогенерация

### Формы MVP

1. **"Купить"** (модалка, вызывается кнопкой "Купить" на карточке товара):
   - Поля: Имя (обяз), Телефон (обяз), Email (необяз), Комментарий (необяз), Товар (скрытое/предзаполненное поле).
2. **"Заявка на дилерство"** (модалка):
   - Поля: Имя (обяз), Телефон (обяз), Email (необяз), Компания (необяз), Комментарий (необяз).
3. **"Обратная связь"** (на странице Контакты и в модалке по кнопке "Заказать" в хедере):
   - Поля: Имя (обяз), Телефон (обяз), Email (необяз), Сообщение (необяз).
4. **"Заказать звонок"** (модалка, кнопка в хедере):
   - Поля: Имя (обяз), Телефон (обяз).
5. **"Оставить отзыв"** (на карточке товара):
   - Поля: Имя (обяз), Текст отзыва (обяз), Рейтинг (необяз, 1–5 звезд).

### Обязательные элементы всех форм

- Чекбокс "Согласен на обработку персональных данных" со ссылкой на страницу `/privacy` (**обязательный**, форма не отправляется без согласия — требование 152-ФЗ).

### Поведение форм

- Клиентская валидация (zod/react-hook-form) + серверная проверка.
- Anti-spam: honeypot-поле + rate-limit на API (не более 5 заявок с одного IP в минуту).
- Успешная отправка: toast "Заявка отправлена, мы свяжемся с вами в ближайшее время".
- Ошибка: понятное сообщение + возможность повтора без потери заполненных данных.

### Канал доставки заявок

- В MVP: отправка на email `asfalt@NovTecAs.ru`.
- Рекомендуется также сохранять заявки в Strapi (`Lead`) для отказоустойчивости.

## 12. Адаптивность и breakpoints

- Сайт должен быть полностью адаптивным (mobile-first подход).
- Breakpoints (Tailwind CSS по умолчанию):
  - `sm`: 640px (мобильные телефоны в ландшафте).
  - `md`: 768px (планшеты).
  - `lg`: 1024px (ноутбуки).
  - `xl`: 1280px (десктоп).
  - `2xl`: 1536px (широкие экраны).
- Ключевые адаптивные поведения:
  - Header: бургер-меню на `< lg`, полная навигация на `>= lg`.
  - Карточки товаров: 1 колонка на `< sm`, 2 на `sm–lg`, 3–4 на `>= lg`.
  - Сайдбар (О компании, Медиа): скрывается на `< md`, показывается как боковая панель на `>= md`.
  - Карта дилеров: сайдбар под картой на `< md`, рядом с картой на `>= md`.
  - Формы-модалки: полноэкранные на `< sm`, по центру экрана на `>= sm`.

### 16.1 Правила миграции "старый сайт -> новый сайт"

- Главная: сохранить структуру "Hero -> Продукция -> Преимущества -> Области применения -> CTA".
- Раздел "О компании" и подпункты: переносим контент в `company/*` с обновлением верстки.
- Раздел "Продукция": переносим карточки, категории, цены, пояснение "уточняйте цену" для позиций с плавающей стоимостью.
- Раздел "Дилеры": переносим логику страницы + форму "Заявка на дилерство".
- Разделы "Новости/Статьи/Видео": объединяются в "Медиа / Полезное" с типизацией в `MediaPost.type`.
- Раздел "Проекты" старого сайта маппится в новый раздел `Портфолио`.
- Разделы "Прайс-лист" и "Доставка" в MVP не выносятся отдельными пунктами меню: контент размещается в релевантных страницах (`Продукция`, `Контакты`) или добавляется после MVP.

### 16.2 SEO-миграция и редиректы

- Обязательно сохранить индексируемость старых URL через `301 redirect` на ближайшие новые страницы.
- Сформировать таблицу соответствий `old_url -> new_url` как отдельный артефакт релиза.
- Для страниц без прямого аналога использовать редирект на родительский раздел, а не на главную.
- После запуска проверить статус-коды, отсутствие `404` на важных старых URL и корректность canonical.

## 17. Критерии приемки MVP (Definition of Done)

- Развертывание проекта через `docker-compose up` без ручных правок.
- Все страницы из раздела "Информационная архитектура" доступны и заполнены тестовым контентом.
- Все формы работают end-to-end (UI -> API -> email и/или запись в `Lead`).
- Товары и публикации создаются/редактируются через Strapi и отображаются на фронте.
